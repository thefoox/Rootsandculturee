## CSS Custom Properties

Declare in `app/globals.css` using Tailwind v4's `@theme` directive. No `tailwind.config.js` needed.

```css
@import "tailwindcss";

@theme {
  /* Brand palette */
  --color-forest:      #1B4332; /* dominant — forest green */
  --color-cream:       #FEFCF3; /* dominant surface — warm white */
  --color-card:        #F5F0E8; /* secondary surface — warm off-white */
  --color-rust:        #A0522D; /* secondary accent — rust red */
  --color-ember:       #B84D00; /* primary accent — burnt orange (WCAG AA compliant) */
  --color-bark:        #5C3D1E; /* tertiary — warm brown, muted text */
  --color-destructive: #C0392B; /* destructive actions only */

  /* Typography */
  --font-heading: 'Merriweather', Georgia, serif;
  --font-body:    'Inter', system-ui, sans-serif;
}
```

**Note on accent color:** The user specified `#CC5500` (burnt orange) but that hex achieves only 4.20:1 against `#FEFCF3` — passes AA large text only. Per WCAG-02 and FOUND-05 requirements, the accent for normal-size interactive text is shifted to `#B84D00` (4.98:1, PASS AA). `#CC5500` may be used for decorative elements (icons ≥ 24px, borders, illustrations) that carry no text.

## Spacing Scale

Declared values (multiples of 4, 8-point grid):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline spacing |
| sm | 8px | Compact element spacing, tight button padding |
| md | 16px | Default element spacing, form field gaps |
| lg | 24px | Card padding, section inner padding |
| xl | 32px | Layout column gaps, card grid gaps |
| 2xl | 48px | Section breaks, hero internal spacing |
| 3xl | 64px | Page-level section spacing |

**Exceptions:**
- Touch targets (mobile): minimum 44px height for all interactive elements (buttons, links, inputs) per WCAG 2.5.5
- Skip-link: positioned `absolute top-0 left-0` off-screen normally; 8px padding when focused
- Focus ring: 2px solid `#1B4332` with 2px offset (total 4px visual gap from element edge)

**Source:** Default 8-point scale; WCAG-01 touch target exception

## Typography

"Kompakt og effektiv" direction from CONTEXT.md D-08. Prioritise information density over airy spacing.

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Body | Inter | 15px | 400 (regular) | 1.5 | Paragraph text, descriptions, form labels |
| Label | Inter | 13px | 500 (medium) | 1.4 | Input labels, table headers, meta text, badges |
| Heading (h3/h4) | Merriweather | 20px | 700 (bold) | 1.25 | Card titles, section subheadings |
| Heading (h1/h2) | Merriweather | 28px | 700 (bold) | 1.2 | Page headings, hero titles |

**Weights in use: 2 (Inter 400/500, Merriweather 700)**

**Source:** CONTEXT.md D-06, D-07, D-08 (compact direction); standard scale defaults

## Color

| Role | Hex | WCAG on Cream | Usage |
|------|-----|---------------|-------|
| Dominant surface (60%) | `#FEFCF3` | — | Page background, main content area |
| Secondary surface (30%) | `#F5F0E8` | — | Cards, sidebar, nav background, modal overlays |
| Forest green | `#1B4332` | 10.78:1 PASS AA | Primary text, nav links, headings, borders |
| Accent — ember (10%) | `#B84D00` | 4.98:1 PASS AA | See reserved list below |
| Rust | `#A0522D` | 5.46:1 PASS AA | Secondary accent, category pills, highlight borders |
| Bark | `#5C3D1E` | 9.54:1 PASS AA | Muted/secondary text, timestamps, captions |
| Destructive | `#C0392B` | 5.29:1 PASS AA | Error messages, delete buttons, form validation errors |

**Accent (`#B84D00`) reserved for:**
1. Primary action buttons ("Logg inn", "Registrer deg", "Tilbakestill passord")
2. Focus ring on interactive elements (2px solid `#1B4332` — forest, not ember)
3. Active/selected nav state indicator (left border or underline)
4. Form submission buttons

**Accent is NOT used for:** body text, headings, placeholder text, borders on non-interactive elements, decorative dividers.

**Dominant/secondary split:** `#FEFCF3` for `<body>` background and `<main>` content area; `#F5F0E8` for `<header>` (sticky nav), cards, modal overlays, form field backgrounds.

**Source:** CONTEXT.md D-01 through D-05; contrast verified via WCAG 2.1 AA calculator 2026-03-30

## Component Inventory (Phase 1 scope)

These are the components the executor must build in this phase. No others.

### Layout Shell
- `<SkipLink>` — absolute positioned, visible on focus, links to `#main-content`
- `<Header>` — sticky, `z-50`, height 64px, contains logo + nav + auth button
- `<MegaMenuNav>` — desktop dropdown revealing category grid with labels
- `<MobileNav>` — hamburger trigger (`☰`), fullscreen overlay with close button (`✕`)
- `<Footer>` — 3-column grid: links / contact info / social media

### Auth Components (modal/overlay pattern per D-10)
- `<AuthModal>` — overlay that mounts on top of current page without route change
- `<LoginForm>` — e-post + passord fields, submit button, "Glemt passord?" link
- `<RegisterForm>` — e-post + passord + fullt navn + adresse fields (D-12), submit button
- `<PasswordResetForm>` — e-post field + submit button

### Shared Primitives
- `<Button>` — variants: primary (ember fill), secondary (forest outline), ghost (no border)
- `<Input>` — text input with label above, error message below via `aria-describedby`
- `<FormError>` — red text block with icon, linked to field via `aria-describedby` (WCAG-05)
- `<Toast>` — sonner-based, `role="status"` for non-critical, `role="alert"` for errors

## Layout Specifications

### Header (sticky)
- Height: 64px
- Background: `#F5F0E8` (secondary surface)
- Bottom border: 1px solid `rgba(27, 67, 50, 0.12)` (forest at 12% opacity)
- Logo: left-aligned, 140px wide max
- Nav links: centered on desktop, hidden on mobile
- Auth trigger: right-aligned, Button variant=ghost, text "Logg inn"
- Hamburger: right-aligned on mobile (≤768px), 44px touch target, `aria-label="Apne meny"`

### Mega-menu dropdown
- Triggered by hover (desktop) + click keyboard Enter/Space on nav item
- Background: `#FEFCF3`, border: 1px solid `rgba(27, 67, 50, 0.15)`
- Box-shadow: `0 8px 24px rgba(0,0,0,0.08)`
- Category grid: 4 columns, each with a 48px icon area + label (13px, weight 500)
- Keyboard: Escape closes, Arrow keys navigate items, Tab moves to next top-level item

### Mobile nav overlay
- Full viewport: `position: fixed; inset: 0; z-index: 100`
- Background: `#1B4332` (forest)
- Text: `#FEFCF3` (cream)
- Links: 24px Merriweather, 700, 48px line height (large touch targets)
- Close button: top-right, 44px, `aria-label="Lukk meny"`, cream X icon

### Auth Modal
- Overlay: `rgba(27, 67, 50, 0.6)` backdrop
- Modal card: `#FEFCF3`, border-radius 8px, max-width 420px, padding 32px
- Trigger behaviour: preserves scroll position, body `overflow: hidden` when open
- Close: Escape key + click outside + explicit close button
- `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to modal heading
- Focus trap: focus moves to first form field on open; returns to trigger on close

### Footer
- Background: `#1B4332` (forest)
- Text: `#FEFCF3` (cream)
- 3-column grid on desktop, single column stacked on mobile
- Padding: 48px top, 32px bottom

## Interaction States (all interactive elements)

| State | Specification |
|-------|---------------|
| Default | As specified in component |
| Hover | Opacity 85% on buttons; underline on text links |
| Focus | 2px solid `#1B4332` outline, 2px offset — applies to ALL interactive elements without exception |
| Active | Scale 0.98, 100ms transition |
| Disabled | Opacity 40%, `cursor: not-allowed`, no hover effect |
| Loading | Button: spinner icon replaces label + `aria-busy="true"`, `disabled` attribute set |
| Error | Border `#C0392B`, `aria-describedby` links field to `<FormError>` message |

**Focus ring MUST be visible on all surfaces.** Forest green `#1B4332` focus ring works on both cream and card surfaces. On the dark forest header/mobile nav/footer, use cream `#FEFCF3` as focus ring instead.

## Accessibility Contract (WCAG 2.1 AA)

These are non-negotiable implementation requirements, not suggestions.

| Requirement | Implementation |
|-------------|---------------|
| WCAG-01 Keyboard nav | All interactive elements reachable via Tab; logical DOM order matches visual order |
| WCAG-01 Focus indicator | 2px solid focus ring, always visible (see Interaction States) |
| WCAG-02 Contrast | All colour pairs verified ≥ 4.5:1 for normal text (see Color section) |
| WCAG-03 Semantic HTML | `<header>`, `<nav>`, `<main id="main-content">`, `<footer>`, `<form>`, `<button>` — never `<div onClick>` |
| WCAG-03 ARIA | `aria-label` on icon-only buttons; `aria-expanded` on toggles; `aria-live="polite"` on toast region |
| WCAG-05 Form errors | `aria-describedby` links every input to its error message element |
| WCAG-06 Skip link | `<a href="#main-content" class="sr-only focus:not-sr-only">` as first child of `<body>` |
| Motion | All transitions wrapped in `@media (prefers-reduced-motion: no-preference)` |
| Language | `<html lang="nb">` on root layout |

**Source:** REQUIREMENTS.md WCAG-01 through WCAG-06, STACK.md WCAG section
